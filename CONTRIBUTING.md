# Contributing

## Scope
This repository is intentionally **no-build** (single HTML file) to keep demos portable.

## Suggested contributions
- Documentation updates (integration notes, architecture diagrams)
- Accessibility improvements
- Refinements to scoring/visualization explanations
- Facilitator guidance

## Process
1. Fork the repo and create a branch.
2. Make focused changes.
3. Open a Pull Request with a clear summary and testing notes.
